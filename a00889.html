template<typename Traits >
const Indices & parafields::DCTDSTFieldBackend< Traits >::localFieldCells ( ) const
inline

Number of entries per dim on this processor.

This is the number of cells per dimension of the extended domain, or the number of cells per dimension for the local part of the extended domain in the case of parallel data distribution. Only half the array per dimension is stored, since the other half contains the same values in reversed order, with the same sign in the case of even symmetry, else with the opposite sign. One additional entry per dimension is needed for the symmetry axis.

Returns
tuple of local cells per dimension

Index parafields::DCTDSTFieldBackend< Traits >::localFieldSize ( ) const

Number of extended field entries stored on this processor.

This is the number of extended field entries that are stored. The backend treats each combination of even and odd symmetry separately, and only ever stores the part that actually needs to be stored. This function returns the number of entries that are needed for this, which is the number of entries when all the domain boundaries have even symmetry.

Returns
number of local degrees of freedom

Define even (DCT) and odd (DST) dimensions.

This function configures the backend to represent extended random fields of a given symmetry in each dimension, either even or odd. It configures the correct FFTW flags, and computes the correct array size for FFTW, since the package handles the zeros in odd dimensions implicitly, which means the data has to be shifted and padding added / removed.

Parameters
typenumber that encodes evenness / oddness in binary

Switch last two dimensions (for transposed transforms)

This function switches the last two dimensions, which is needed for FFTW transposed transforms, where the Fourier transform of the matrix is stored transposed to eliminate the final transpose step. Is automatically called by the transform methods, but may be needed when a newly created backend should be constructed directly in frequency space.
